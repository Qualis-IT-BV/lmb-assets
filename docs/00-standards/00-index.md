Deze map bevat de **standaard werkwijzen en afspraken** die gelden voor **alle Qualis IT projecten**.

- versioning
- branching
- releases

---

## Gebruik van deze standaarden

- Deze standaarden zijn **project-onafhankelijk**
- Project-specifieke informatie hoort **niet** in deze map
- Afwijkingen worden vastgelegd in `docs/10-project/`

Wijzigingen aan deze standaarden:
- gebeuren bewust
- via Pull Requests
- met review

---

## Overzicht standaarden

- `versioning.md`  
  Definieert hoe releases en builds worden benoemd.

- `branching.md`  
  Beschrijft de standaard branching-strategie.

- `releases.md`  
  Beschrijft het releaseproces en promotion flow.

- `documentation-rules.md`  
  Borgt hoe documentatie wordt beheerd en aangepast.

- `changelog-policy.md`  
  Regels voor het bijhouden van changelogs.

- `security-privacy.md`  
  Basisprincipes voor security en privacy.

## Belangrijk

Deze documenten zijn **leidend**.

- expliciet

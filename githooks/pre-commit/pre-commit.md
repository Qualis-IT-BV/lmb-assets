
Header moet minimaal uit deze onderdelen bestaand:
Indien dat niet zo is worden deze toegevoegd.

Project:
    - De naam van het project/repo

Component:
    - Naam van het component

Build:  
    - Is de build volgens de standaarden.
    - Er wordt ook gecontroleerd of de dag nog klopt.
    - Indien de dag niet juist is, wordt deze ook aangepast en wordt de teller weer op  0 gezet.

First Release:
    - De versie van de eerste release dat het component is verschenen.

Last Change:
    - De release dat deze voor het laatst is aangepast en datum Release.

Source:
    - Van waar is dit component afgeleid? Bij nieuwe code (New)
    - Indien bestaand Dus indien versie nummer aanwezig dan:
        Filename Release x.y.z Version/Build: x.y.z (van het component) 


Purpose:
    - Doel van de release (In te vullen door de ontwikkelaar) Zie onderstaand voorbeeld
